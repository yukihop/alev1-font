# 目的・要件整理と実装案の概略

## 目的

オリジナルのフォントを作成・配布したい。このフォントは通常の文字フォントではなく、**256個の概念グリフを持つシンボルフォント**として設計する。

各グリフは、デジタル数字の7セグメント表示のように **8個の独立したコンポーネント（部首的なもの）** の ON/OFF の組み合わせで構成される。したがって表現可能なグリフ数は `2^8 = 256` であり、各グリフは 8bit のビットパターンに一意対応する。

さらに、各グリフは本質的に漢字のような表意文字である。各文字に `"love"` や `"perform"` のような**意味**が割り当てられており、Material Symbols のように、入力した語をリガチャベースで1グリフに変換する形で動作する。

加えて、概念語だけでなく、**16進2桁の大文字表記でも各グリフを直接指定**できるようにしたい。つまり `00` から `FF` までの 256 通りを、それぞれ対応する1文字へ ligature で変換する。概念語と 16 進表記の両方から同じグリフへ到達できる。

例:

- `love` → 概念グリフ1文字
- `i learn human` → グリフ3文字
- `B1` → `0xB1` に対応する概念グリフ
- `CA` → `0xCA` に対応する概念グリフ

## 要件

### 機能要件

- 8個のコンポーネントの ON/OFF により 256 グリフを表現する
- 各グリフはシンプルな幾何学形状の合成であり、**プログラムで記述可能**である
- 各グリフに0個以上の意味語が対応付いている（0の場合もある）
- 意味語を入力すると、OpenType の ligature 機能で対応グリフへ変換される
- `00`〜`FF` の **16進2桁表記**でも各グリフを指定できる
- 最終的に **TTF** と **WOFF2** を生成して配布する

### 開発要件

- GUI 中心ではなく、**コードと設定ファイルを正本**として管理
- ビルドは GitHub Actions で安定的に回せる構成が望ましい
- 可能な限り **JavaScript / Node.js 中心**で作業するが、安定ビルドのために部分的に Python (with uv) を導入してよい。

# 実装案の概略

- 開発の正本は **YAML + Node.js** にする
- Node.js で 8コンポーネントから 256 グリフを自動生成する
- Node.js で ligature 用の `.fea` を自動生成する
- 中間ソースは **UFO + `.fea`**
- 最終ビルドは **`fontmake` を CLI として使用**
- 配布物は **TTF + WOFF2**
- 内部コードポイントは **PUA (`U+E000`〜`U+E0FF`)** を使う想定

## 技術選定

- **Node.js / JavaScript**: 日常の開発の中心。YAML を読み、256 グリフと feature 定義を生成する
- **YAML**: コンポーネント定義、概念語対応、フォント設定の正本
- 中間ファイル（Git 管理外でいいならそうする）
  - **UFO**: グリフ輪郭・メトリクス・コードポイントを `fontmake` に渡す
  - **`.fea`**: `love` → 概念グリフ、`B1` → `0xB1` のグリフ、のような ligature ルールを定義
- **`fontmake`**: UFO + `.fea` から TTF を作る最終ビルドツール。安定ビルド重視のため採用
- **Python 実行環境**: `fontmake` を使う以上必要。ただし日常の制作コードは Python でなくてよい

## 一言でまとめると

**YAML + Node.js で 256 グリフと ligature 定義を生成し、UFO + `.fea` を `fontmake` でビルドして TTF/WOFF2 を作る。**
