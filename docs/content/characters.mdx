## ALEV文字一覧

上位4ビットを左（行）、下位4ビットを上（列方向）に並べたすべての文字の一覧です。

上位4ビットは外縁の4つの部品 :0x80: :0x40: :0x20: :0x10: に対応しており、その組み合わせが大まかな品詞を決定しています。

| 上位4ビット（左の列） | 形                          | 品詞                                       |
| --------------------- | --------------------------- | ------------------------------------------ |
| 0000                  |                             | 英語でいう接続詞・前置詞系                 |
| 0001                  | :0b00010000:                | 数字                                       |
| 0100                  | :0b01000000:                | 人称名詞・固有人名（ロボットもここに含む） |
| 0110                  | :0b01100000:                | 指示名詞（「これ」「それ」）               |
| 0111                  | :0b01110000:                | 抽象名詞（「何か」）                       |
| 1000                  | :0b10000000:                | 助動詞・疑問詞・否定辞などの機能語         |
| 1010, 1011            | :0b10100000: & :0b10110000: | 動詞                                       |
| 1100                  | :0b11000000:                | 一般名詞                                   |
| 1101                  | :0b11010000:                | 形容詞                                     |

黄色でハイライトされているものは公式に登場が確認されているものです。出典については[全文コーパス](/corpus)を参照してください。ほとんどの文字は意味と品詞が判明しています。

## ALEV-1の基本文法について

- 基本的には、英語や中国語に似たSVO言語です。「主語＋動詞＋目的語」が基本パターンとなっています。
- 動詞には時制はなく、基本的に「～しました」と「～します」の違いは文脈に依存します。ただしコード上で隣り合っている :prf: が「～済みです」のようなアスペクトを、:0x86: が「～するつもりです」のような意図を表しているようで、これらが間接的に時制を表しているとも言えます。この辺りは中国語に似ています。
- 否定語 :not: はコピュラ、動詞、名詞をいずれも否定することができ、常に前置されます。"is not" ではなく "not is" になるイメージ。
- 疑問文は語順を変えることではなく、:interrogative: を先頭に置くことで表現します。たとえば、:straylight seek perfection: が「ストレイライトは完璧を求める」なら :interrogative straylight seek perfection: で「ストレイライトは完璧を求めます**か**」になる、という具合です。日本語の「か」と同じ役目の語が文頭に来るのだと考えてください。
- 命令文は確認されていません。
- 形容詞や動詞の明示的な名詞化 (nominalization) ルールは存在しないようです。つまり文脈によって :perfect: は「完璧な」という意味の形容詞になったり「完璧さ」という意味の名詞になったりします。「完璧に」のような副詞が一切確認されていないため、これでもなんとか成立しているようです。
- 基本的に純粋な表意文字であり、たとえば :fuyu: が「ふゆ」なのか「冬優子」なのかは区別されません。唯一の例外が :wanchan: です。
